Nginx 配置优化

  • nginx

前言

　　Nginx 中有非常丰富的参数来帮助我们根据项目的实际情况优化 nginx 的性能。

全局配置

user 决定了 nginx 的 worker 运行的权限，一般我们不希望 nginx 的 worker 拥有 root 权限。
worker_processes 定义了 nginx 对外提供服务时的 worker 进程数。通常我们会设置成 auto，nginx 会启动和 CPU 核心数量相同的 worker。
worker_rlimit_nofile 定义了 worker 能够打开的文件描述符的数量，如果没有设置则会使用 ulimit -n 返回的数量。过小的值会导致新连接建立是返回 too many open files 错误。

Event 模块

worker_connections 定义了 worker 进程允许的同时打开的最大连接数。
multi_accept 如果关闭这个选项 nginx 主进程一次只能接收一个连接。

Http 模块

server_tokens 隐藏版本号
sendfile 指定是否使用 sendfile 系统调用来传输文件。sendfile 系统调用在两个文件描述符之间直接传递数据，完全在内核中操作，从而避免了数据在内核缓冲区和用户缓冲区之间的拷贝，操作效率很高，被称之为零拷贝。
tcp_nopush 在 unix 上影响 TCP_NOPUSH，在 linux 上影响 TCP_CORK，TCP_CORK 选项可以提高网络的利用率，因为它直接禁止了小包的发送。注意 Nagle 算法没有禁止小包发送，只是禁止了大量小包的发送。
tcp_nodelay 影响 TCP_NODELAY 设置将关闭 Nagle 算法。
limit_conn_zone 使用 $binary_remote_addr 作为 key，配合 limit_conn 限制同一个 IP 最大连接数。
